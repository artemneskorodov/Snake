### Суть механики:
После смерти змеи ее тело превращается в блок костей.

### Требования к реализации:
1. Количество костей оставленных змеей - первые несколько блоков змеи. Это количество должно быть настраиваемым параметром, должна быть также возможность задать константой, чтобы вся змея становилась
блоком костей, или чтобы какой-то процент блоков змеи становились костями при смерти.

2. Время жизни обломков также должно быть константой, должна быть возможность поставить бесконечное значение.

3. Свойство костей - непроходимость. Интелектуальные боты должны учитывать это и обходить препятствия.

### Что это принесет в игру:
1. Данная механика постепенно превращает игровое поле в запутанный лабиринт, который особенно сильно выделит отличие тупых ботов и интелектуальных.

2. К финалу игры поле будет представлять из себя хаотичный набор костей, это позволит сделать игру достаточно конечной по времени и более динамичной.

3. В будующем это позволит добавлять интересные механики ботов: например, можно добавить пугливых интелектуальных ботов, которые будут избегать столкновений и прятаться, а при наличии костей они будут прятаться за укрытиями.
